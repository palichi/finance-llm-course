## 🔍 RAG 동작 원리 (만들기 전에 먼저 이해하기)

```
1. 질문 입력
   "삼성전자 12월 주가는?"

2. 질문 → 벡터 변환
   [0.123, -0.456, ...] (1536차원)

3. ChromaDB에서 유사 문서 검색 (Top-K=5)
   "2024년 12월 31일 삼성전자 종가 78,500원..."
   "2024년 12월 30일 삼성전자 종가 79,000원..."
   ...

4. 검색된 문서 + 질문 → GPT에 전달
   "다음 데이터를 참고해서 질문에 답해줘:
    [검색 결과]
    질문: 삼성전자 12월 주가는?"

5. GPT 답변 생성
   "수집된 데이터에 따르면 삼성전자의 12월 종가는
    평균 78,750원이며..."
```

> 💡 RAG는 수집된 데이터 안에서만 답합니다.
> 데이터에 없는 질문은 "데이터에 없다" 고 솔직하게 답하는 게 정상입니다.

## 💡 이 실습의 핵심 포인트

```
rag_chain.py  → Claude Code로 생성 (RAG 체인 구성 원리 학습)
rag_app.py    → Claude Code로 생성 (채팅 웹앱으로 시각화)

일반 LLM 챗봇과 RAG 챗봇의 차이를 직접 체감하는 것이
이 실습의 목표입니다.

일반 LLM: 학습 데이터 기준 → 오래되거나 부정확한 답변
RAG:      수집한 실제 데이터 기준 → 정확한 수치와 날짜 답변
```

## DBMS와 RAG 차이

DBMS           (관계형 데이터베이스)             RAG (Retrieval-Augmented Generation)
──────────────────────────────────────────────────────
저장 방식      구조화된 테이블 (행/열)            벡터(임베딩) — 의미를 숫자로 변환해 저장
검색 방식      정확한 조건 일치 (exact match)     의미적 유사도 (semantic similarity)
질문 형태      "정형화된 질의" (SQL)              "자연어 질문" 그대로
결과           원본 데이터 그대로 반환            관련 문서를 찾아 LLM에게 전달 → LLM이 답 생성



핵심 차이 한 줄
구분   DBMS                          RAG
검색  기준정확히 일치 (=, >, LIKE 등) 의미적으로 유사 (벡터 거리)
질의  언어SQL (정형 문법)             자연어 (비정형)
강점  정형 데이터의 정밀 집계          비정형 데이터의 맥락 이해
결과  데이터 그 자체                   LLM이 재구성한 답변
