## 🔍 3가지 방식 한눈에 비교

```
방식 1 · 일반 LLM (인터넷 학습 데이터 그대로)
질문 → GPT 학습 데이터 기준으로 즉답
"제가 학습한 시점 기준으로는..." (부정확하거나 오래된 정보)

방식 2 · RAG (검색 증강 생성)
질문 → ChromaDB 검색 → 검색된 데이터 + 질문 → GPT 답변
"수집한 실제 데이터에 따르면..." (정확하지만 검색 범위 안에서만)

방식 3 · Fine-tuning (모델 자체를 재학습)
질문 → 금융 도메인으로 재학습된 모델이 직접 답변
"모델 자체가 금융 용어와 패턴을 학습함" (느낌·어투까지 도메인화)
```

---

## 비유로 이해하기

```
일반 LLM   = 외부 전문가에게 전화로 묻기 (배경지식은 있지만 최신 자료 없음)
RAG       = 자료를 손에 들고 찾아가며 답하는 사람 (정확하지만 자료 범위 안에서만)
Fine-tuning = 해당 분야를 전문적으로 공부한 사람 (말투·직관까지 도메인화되지만 비용·시간 ↑)
```

---

## 비교표

| 항목        | 일반 LLM           | RAG                     | Fine-tuning                           |
|------------|--------------------|-------------------------|---------------------------------------|
| 데이터 출처 | 인터넷 학습 데이터  | 직접 수집한 ChromaDB     | 직접 학습시킨 모델 가중치               |
| 최신 정보   | ❌ 학습 시점까지만 | ✅ 수집한 만큼 최신      | ❌ 학습 시점까지만                     |
| 정확한 수치 | ❌ 부정확/추측 가능 | ✅ 정확 (출처 표시 가능) | 🔶 패턴은 학습, 수치는 부정확할 수 있음|
| 구축 비용   | 없음 (API 비용만)   | 낮음 (임베딩 비용)       | 높음 (GPU, 학습 시간)                  |
| 구축 시간   | 즉시 사용 가능      | 수 분 (DB 구축)          | 수십 분~수 시간 (학습)                 |
| 적합한 용도 | 일반 지식 질문      | 최신·정확한 데이터 조회   | 도메인 말투·패턴 학습                  |

### STEP 5 · 비교 테스트 — 질문 유형별로 결과가 다름을 확인

#### 테스트 1 · 데이터에 있는 질문 (RAG가 강함)
```
질문: "삼성전자 12월 주가는?"

예상 결과:
🌐 일반 LLM   → "구체적인 수치는 알 수 없습니다" 또는 추측성 답변
🔍 RAG        → "2024년 12월 31일 종가 78,500원..." (정확한 수치)
🎯 Fine-tuning → 검색 정확도가 RAG보다 미세하게 높거나 비슷
```

#### 테스트 2 · 데이터에 없는 최신 질문 (RAG의 한계 확인)
```
질문: "오늘 코스피 지수는?"

예상 결과:
🌐 일반 LLM   → 모르거나 오래된 정보로 추측
🔍 RAG        → "수집된 데이터에 코스피 지수 정보가 없습니다"
              (솔직하게 답하는 것이 정상! RAG는 거짓말 안 함)
🎯 Fine-tuning → 마찬가지로 데이터에 없으면 답 못함
```

#### 테스트 3 · 일반 지식 질문 (일반 LLM이 강함)
```
질문: "RSI 지표가 뭐야?"

예상 결과:
🌐 일반 LLM   → 정확하고 일반적인 설명 (학습 데이터에 풍부함)
🔍 RAG        → 수집 데이터에 RSI 설명이 없으면 답 못하거나
              관련 없는 검색 결과를 가져옴
🎯 Fine-tuning → 일반 지식 질문에는 한계
```

## 💡 이 실습의 핵심 포인트

```
"어느 방식이 무조건 좋다"가 아니라
"질문 유형에 따라 적합한 방식이 다르다"는 것을 이해하는 것이
이 실습의 목표입니다.

최신·정확한 데이터 질문   → RAG 가 적합
일반 지식·설명 질문       → 일반 LLM 이 적합
도메인 특화 말투/패턴 필요 → Fine-tuning 이 적합

실무에서는 보통 RAG + 일반 LLM 을 함께 쓰고,
Fine-tuning 은 비용이 커서 꼭 필요한 경우에만 사용합니다.
```
