## 0-1. 가장 먼저 — 우리는 "무엇을 하기 위해" 이 모델을 학습시키는가?

모든 데이터 전처리·설계 결정에 앞서, 가장 먼저 답해야 하는 질문입니다.

```
최종 목적
        │
        ▼
"주식 종목 하나의 일별 가격·거래량 데이터를 보고,
 지금 사야 할지 / 팔아야 할지 / 들고만 있어야 할지를
 스스로 판단하는 자동매매 에이전트를 만든다"
```

즉, 이 모델은 **가격을 예측하는 모델이 아니라, "지금 행동을 뭘 해야 하는지"를 결정하는 모델**입니다.
("내일 가격이 얼마일까" ❌  →  "지금 사면/팔면/들고 있으면 수익이 나는가" ✅)

### 왜 예측 모델이 아니라 강화학습(PPO)인가

```
일반적인 AI 모델 (지도학습)              PPO (강화학습) 에이전트
─────────────────────────────────────
"정답"이 주어진 데이터로                  "정답" 없이, 직접 행동해보고
정답을 맞추도록 학습                      그 결과(수익/손실)로 배움

예: 내일 가격이 얼마일지 예측              예: 지금 사보고, 그 결과(보상)로
                                             "잘 산 것"이 무엇인지 스스로 학습
```

매매라는 행위는 "미리 정해진 정답 라벨"이 없습니다 — 언제 사는 게 정답인지는 사후적으로만 알 수 있습니다.
 그래서 직접 행동하고 그 결과(보상)로 배우는 **강화학습** 구조가 이 문제에 맞습니다.

### 이 목적이 이후의 모든 설계 결정을 결정짓는다

| 설계 결정 | 위 목적과의 연결 |
|------------------------------|--------------------------------------------------------------------------|
| 종목별로 쪼개서 학습          | 한 종목의 가격 흐름 안에서만 "매매 타이밍"을 배우게 하려고                   |
| 15개 컬럼 중 9개 제외         | 종목명·시가총액 같은 정보는 "언제 사고 팔지" 판단에 직접 도움이 안 되기 때문 |
| open/high/low/close/volume   | 매매 타이밍 판단에 필요한 최소한의 가격·거래량 정보이기 때문                |
| 행동(Action) = 보유/매수/매도 | 최종적으로 에이전트가 내려야 하는 결정 그 자체                              |
| 보상(Reward) = 수익+/손실−    | "잘 산 것"과 "잘 판 것"이 무엇인지를 결과로 가르치기 위해                   |

**한 문장 요약: 이 문서에서 다루는 컬럼 선택 작업은 결국 "과거 가격 흐름만 보고 매매 타이밍을 스스로 판단하는 자동매매 에이전트를 만들기 위한" 데이터 준비 과정이다.**

## 0-2. 그 다음 — "무엇을 학습시킬 것인가?" (입력 데이터의 관점)

전처리를 설계하기 전에 먼저 정해야 하는 것은 **"에이전트에게 무엇을 가르칠 것인가"** 입니다.

```
이번 PPO 에이전트의 학습 목표
        │
        ▼
"하루하루의 가격·거래량 흐름을 보고,
 지금 사야 할지 / 팔아야 할지 / 들고 있어야 할지를 판단하는 것"
        │
        ▼
즉, 학습 대상은 ── "가격의 움직임" 그 자체이다.
        (기업 정보, 시장 분류, 시가총액 등 "맥락 정보"가 아니다)
```

이 목표 한 줄이 **15개 → 6개로 컬럼을 줄이는 모든 선택의 기준**이 됩니다.

---

## 1. 원본 15개 컬럼 — 한국거래소(금융위원회) 표준 필드

```
data/stock_prices.csv  (원본, 15개 컬럼 그대로 보존)
┌─────────────┬──────────────────┐
│  필드명       │  의미             │
├─────────────┼──────────────────┤
│  basDt       │  기준일자          │
│  srtnCd      │  단축코드(종목코드)  │
│  isinCd      │  ISIN코드         │
│  itmsNm      │  종목명            │
│  mrktCtg     │  시장구분          │
│  clpr        │  종가             │
│  vs          │  전일대비          │
│  fltRt       │  등락률           │
│  mkp         │  시가             │
│  hipr        │  고가             │
│  lopr        │  저가             │
│  trqu        │  거래량           │
│  trPrc       │  거래금액          │
│  lstgStCnt   │  상장주식수        │
│  mrktTotAmt  │  시장총액(시가총액) │
└─────────────┴──────────────────┘
```

API가 제공하는 필드는 15개 전부지만, **모델이 풀어야 할 문제(매매 타이밍 판단)에 필요한 정보는 그중 일부뿐**입니다.

## 2. 분류 기준 — 15개를 4가지 역할로 나눠보기

```
                         15개 원본 컬럼
                              │
        ┌─────────────┬───────┴───────┬─────────────┐
        ▼             ▼               ▼             ▼
   ① 식별자        ② 가격(OHLC)    ③ 거래량      ④ 파생·맥락 정보
  srtnCd, isinCd    mkp, hipr,       trqu         vs, fltRt, trPrc,
  itmsNm, mrktCtg   lopr, clpr                    lstgStCnt, mrktTotAmt
   (4개)             (4개)           (1개)          (5개)

  + basDt (기준일자, 1개) → 날짜는 식별·정렬용으로 별도 보존

  ─────────────────────────────────────────────────────
  4개 + 4개 + 1개 + 5개 + 1개(날짜) = 15개
```

| 역할           | 컬럼                                    | 학습에 쓰는가?                           |
|----------------|----------------------------------------|-----------------------------------------|
| 날짜           | basDt                                  | ✅ 보존 (정렬·인덱스용, 신경망 입력 아님) |
| 가격(OHLC)     | mkp, hipr, lopr, clpr                  | ✅ 사용 (open/high/low/close)           |
| 거래량         | trqu                                   | ✅ 사용 (volume)                        |
| 식별자         | srtnCd, isinCd, itmsNm, mrktCtg        | ❌ 제외                                 |
| 파생·맥락 정보 | vs, fltRt, trPrc, lstgStCnt, mrktTotAmt| ❌ 제외                                 |

---

## 3. 왜 9개를 빼는가 — 컬럼별 이유

```
        [ ❌ 제외되는 9개 ]
        ─────────────────────────────────────────────
        isinCd       ──▶ 종목 식별용 코드. srtnCd와 중복 정보. 가격 패턴과 무관.
        itmsNm       ──▶ 종목명(문자열). 신경망 입력이 될 수 없고, 종목 분리는
                          이미 "파일 단위 분리"로 처리되어 불필요.
        mrktCtg      ──▶ 코스피/코스닥 구분. 종목마다 고정값 → 시계열 신호 없음.
        vs           ──▶ 전일대비(가격차). close에서 바로 계산 가능 → 중복 정보.
        fltRt        ──▶ 등락률(%). 역시 close의 파생값 → 중복 정보.
        trPrc        ──▶ 거래금액(= 가격 × 거래량). trqu·close의 파생값.
        lstgStCnt    ──▶ 상장주식수. 거의 변하지 않는 정적 값 → 매매 타이밍과 무관.
        mrktTotAmt   ──▶ 시가총액(= 가격 × 상장주식수). 역시 파생값.
```

**공통된 제외 이유는 두 가지로 요약됩니다.**

1. **중복 정보(파생값)** — `vs`, `fltRt`, `trPrc`, `mrktTotAmt`는 이미 사용하는 가격·거래량으로부터 계산 가능한 값입니다. 같은 정보를 두 번 넣으면 학습에 잡음만 추가됩니다.
2. **정적 정보(시계열 신호 없음)** — `isinCd`, `itmsNm`, `mrktCtg`, `lstgStCnt`는 같은 종목 안에서 거의 변하지 않는 값입니다. 매수/매도 "타이밍"을 가르치는 데는 의미가 없습니다.

---

## 4. 왜 5개(+날짜)는 남기는가

```
        [ ✅ 남기는 5개 + 날짜 ]
        ─────────────────────────────────────────────
        basDt  ──▶ 시계열 순서 보존을 위한 기준 (모델 입력 아님, 정렬·검증용)
        open   ──▶ 당일 시작가 — 갭 상승/하락 패턴
        high   ──▶ 당일 최고가 — 저항선·돌파 패턴
        low    ──▶ 당일 최저가 — 지지선·하락 방어 패턴
        close  ──▶ 당일 마감가 — 가장 중요, 다음날 시가의 기준
        volume ──▶ 당일 거래량 — 가격 움직임의 신뢰도 판단
```

이 5개는 서로 **중복되지 않으면서**, 하루의 가격 움직임을 빠짐없이 설명하는 **최소 구성(OHLCV)**입니다. 금융 시계열 모델링에서 표준적으로 쓰이는 조합이기도 합니다.

---

## 5. 전체 흐름 정리

```
[ 15개 원본 컬럼 ]
        │
        ▼
  (1) 식별자 4개 제거 ──▶ 종목 구분은 "파일 단위 분리"로 이미 해결됨
  (2) 파생값 4개 제거 ──▶ vs, fltRt, trPrc, mrktTotAmt = 이미 갖고 있는 값의 계산 결과
  (3) 정적값 1개 제거 ──▶ lstgStCnt = 시계열 신호 없음
        │
        ▼
[ 6개만 남김: basDt, open, high, low, close, volume ]
        │
        ▼
  basDt → 정렬/검증용으로 보존 (모델 입력 아님)
  open, high, low, close, volume → 실제 PPO 신경망 입력 (State)
```

## 6. 결론 — 설계 의도 요약

```
"무엇을 학습시킬 것인가?"
        │
        ▼
"가격·거래량의 시계열 흐름에서 매매 타이밍을 배우게 한다"
        │
        ▼
이 목표에 직접 기여하지 않는 컬럼(식별자·파생값·정적값) 9개는
        │
        ▼
   모델에 잡음(noise)만 추가하므로 제거한다
        │
        ▼
결과: 원본 15개 컬럼 → 학습에 필요한 6개(날짜 1 + 입력 5)로 정제
```

즉, 9개를 빼는 이유는 "정보가 부족해서"가 아니라 **"학습 목표(매매 타이밍 판단)에 직접 기여하지 않는 중복·정적 정보를 걸러내기 위해서"**입니다.

---

*문서 요약: 컬럼 선택은 데이터가 적어서가 아니라, 먼저 정한 학습 목표(가격 흐름 기반 매매 타이밍 학습)에 맞춰 "필요한 정보"와 "불필요한 정보"를 가려낸 결과다. 15개 중 6개(날짜+OHLCV)만 남기고 9개(식별자 4 + 파생값 4 + 정적값 1)를 제거하는 것이 그 결론이다.*
