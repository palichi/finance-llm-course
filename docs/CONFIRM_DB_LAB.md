## STEP 3 · 아래 프롬프트를 그대로 붙여넣기

```
confirm_db.py 파일을 만들어줘.

이 파일은 ChromaDB에 데이터가 잘 저장됐는지
Streamlit으로 시각적으로 확인하는 앱이야.

기능 1 · 상단 지표 카드 3개
  - 저장된 문서 수 (col.count())
  - 임베딩 차원 수 (1,536)
  - 유사도 방식 (코사인)

기능 2 · 저장된 문서 샘플 보기
  - col.peek(limit=5) 로 5건을 가져와
  - 각 문서를 st.expander 로 접을 수 있게 표시
  - expander 제목: "[날짜] 종목명 (종목코드)"
  - expander 안에: 문서 내용(st.text) + 메타데이터(st.json)

기능 3 · 실시간 검색 테스트
  - st.text_input 으로 질문 입력창 만들기
  - 검색 버튼 클릭 시:
    1. 질문을 OpenAI 임베딩으로 벡터 변환
    2. ChromaDB에서 유사도 상위 5건 검색
    3. 각 결과를 st.expander 로 표시
       제목: "1위 · 유사도 0.923 · [날짜] 종목명"
       내용: 문서 전체 텍스트

기타 조건:
  - chroma_db 폴더가 없으면 st.error 로 안내 후 st.stop()
  - .env 파일은 ../../.env 경로에서 읽기
  - ChromaDB 컬렉션명: "stock_data"
  - OpenAI 임베딩 모델: text-embedding-3-small
  - 파일명: confirm_db.py
```

## ❓ 자주 발생하는 오류

**오류: `ValueError: Collection stock_data does not exist`**
```bash
# build_db.py 를 먼저 실행
python build_db.py
```

**오류: `AuthenticationError` (OpenAI)**
```
.env 파일의 OPENAI_API_KEY 확인
```

**오류: 포트 충돌**
```bash
streamlit run confirm_db.py --server.port 8502
```

**앱은 뜨는데 검색 결과가 없어요**
```
질문을 더 구체적으로 바꿔보세요
예: "주가" → "삼성전자 종가 높은 날"
```

## 💡 이 실습의 핵심 포인트

```
1. Claude Code에 기능을 "말로 설명" 하면 코드가 만들어진다
2. 만들어진 코드를 바로 실행해서 결과를 확인한다
3. 안 되는 부분은 Claude Code에 다시 물어보면 된다

→ 이것이 바이브코딩의 핵심 사이클입니다.
```
