```
day1/05_trading/app.py 파일을 만들어줘.

KIS OpenAPI 모의투자로 주식을 매수/매도하는
Streamlit 주문 웹앱이야. 트레이딩 스킬이 설치되어 있다면
해당 스킬의 KIS API 호출 규칙을 우선 참고해서 만들어줘.

기본 설정:
  .env 파일: ../../.env 경로에서 읽기
  KIS_APP_KEY, KIS_APP_SECRET, KIS_ACCOUNT_NO 읽어오기
  모의투자 Base URL: https://openapivts.koreainvestment.com:29443
  페이지 제목: "💰 모의투자 주문"

기능 1 · 종목 조회 영역 (상단)
  종목코드 입력창 + 조회 버튼
  TR: FHKST01010100 (현재가 조회)
  현재가, 전일대비, 등락률을 st.metric 으로 표시
  상승: 빨간색 / 하락: 파란색

기능 2 · 호가창 (좌측 컬럼)
  TR: FHKST01010200 (호가 조회)

  매수호가 5단계 (아래로 낮은 가격)
  각 호가의 잔량도 함께 표시
  st.dataframe 또는 직접 만든 테이블 스타일로 표시

기능 3 · 주문 입력 영역 (우측 컬럼)
  st.tabs 로 "매수" / "매도" 구분
  공통 입력: 주문수량(number_input), 주문가격(number_input)
  "현재가로 입력" 버튼 → 주문가격에 현재가 자동 채움
  주문 방식 선택(selectbox): 보통가 / 시장가

    TR: VTTC0802U (모의 매수 주문)
    "매수 주문" 버튼 클릭 시 st.confirm 또는
    체크박스("주문 내용을 확인했습니다")로 2단계 확인 후 제출

  매도 탭:
    TR: VTTC0801U (모의 매도 주문)
    보유수량을 잔고조회(VTTC8434R)로 먼저 가져와서
    주문수량이 보유수량을 넘지 않도록 검증
    "매도 주문" 버튼도 동일하게 2단계 확인 절차 적용

기능 4 · 주문 결과 표시
  주문 성공 시 st.success 로 주문번호 표시
  주문 실패 시 st.error 로 실패 이유 표시 (KIS 응답 메시지 그대로)
```

## 📐 주문 API 핵심 구조

```
[현재가/호가 조회] → 가격 확인
        ↓
[주문 정보 입력] → 종목, 수량, 가격, 매수/매도 구분
        ↓
[주문 TR 호출] → VTTC0802U(매수) / VTTC0801U(매도)
        ↓
[응답 확인] → rt_cd "0"=성공 / msg1 메시지 확인
```

### 주문 TR_ID (모의투자)
```
TR ID            설명
─────────────────────────────
VTTC0802U       매수 주문
VTTC0801U       매도 주문
VTTC8036R       정정취소가능주문조회
VTTC0803U       주문 정정/취소
VTTC8434R       잔고 조회
```

### 주문 구분 코드 (ORD_DVSN)
```python
ORD_DVSN = "00"  # 보통가(지정가)
ORD_DVSN = "01"  # 시장가
```

### 주문 응답 확인 패턴
```python
res = response.json()
if res["rt_cd"] == "0":
    order_no = res["output"]["ODNO"]
    st.success(f"주문 성공: {order_no}")
else:
    st.error(f"주문 실패: {res['msg1']}")
```

## ❓ 자주 발생하는 오류

**주문 거부 (rt_cd != "0")**
```
원인: 장 운영시간 외 주문 / 잔고 부족 / 종목코드 오류
해결: 09:00~15:30 사이에 테스트
     보유수량 확인 후 매도 주문
     종목코드 6자리 확인 (앞자리 0 누락 주의)
```

**Hash Key 오류**
```
원인: 주문 API는 본문(body) 해시값(Hash Key)이 필요한 경우가 있음
해결: Claude Code 에 요청
"주문 함수에 Hash Key 발급 단계를 추가해줘.
 TR: hashkey API 먼저 호출해서 받은 값을
 주문 요청 헤더에 포함해줘."
```

**잔고가 바로 갱신되지 않아요**
```
원인: 모의투자 체결은 약간의 지연이 있을 수 있음
해결: 주문 후 몇 초 뒤 "잔고 새로고침" 버튼을
     다시 눌러서 확인
```

**계좌번호 형식 오류**
```
원인: KIS_ACCOUNT_NO 는 "계좌번호(8자리)-상품코드(2자리)"로
     분리해서 API에 전달해야 함
해결: .env 의 계좌번호를 CANO(8자리)와
     ACNT_PRDT_CD(2자리)로 나눠서 사용
```
